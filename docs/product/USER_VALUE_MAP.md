# User Value Map

## Primary Value

Martenweave reduces model drift by turning scattered model knowledge into validated, traceable, reviewable model files.

## User Problems

| Problem | Martenweave value |
|---|---|
| Data dictionaries live in spreadsheets and tickets | Canonical objects with stable IDs and references. |
| Columns are not linked to business meaning | `Attribute` and `FieldEndpoint` separation. |
| Mappings and rules are hard to audit | Trace, lineage, impact, and generated indexes. |
| Dataset gaps are found too late | Profiling and gap detection against model knowledge. |
| AI suggestions are risky | PatchProposal first, validation, human approval. |
| Changes lack rationale | Issues, decisions, evidence, ChangeRequests, audit events. |

## Buyer-Neutral Scope

The same core supports migration teams, data governance teams, analytics model owners, product data owners, and platform engineers. Domain packs tailor examples and rules.
