# MVP Boundary

## MVP Sentence

Martenweave Core proves that data model knowledge can be stored as canonical files, deterministically validated, indexed for search/impact, and safely updated through proposal-first AI assistance.

## In Scope

- initialize a model repository;
- load canonical model files;
- validate objects, IDs, types, references, ownership, value lists, and domain-pack rules;
- build disposable SQLite/search/lineage indexes;
- profile local datasets and detect model gaps;
- show trace and impact for fields, attributes, mappings, rules, and ownership;
- create issue drafts and patch proposals;
- apply changes only after review and approval;
- export model views for review.

## Out Of Scope

- generic chatbot;
- workflow engine;
- direct writes to external business systems;
- Neo4j, Kafka, SaaS tenancy, or heavy platform infrastructure;
- UI as an MVP dependency;
- cloud dependency in core;
- making any starter domain define the core model.

## Starter Domains

SAP Business Partner and product catalog examples are proof scenarios. They demonstrate how domain packs can add examples and validation extensions. They do not define MVP scope.
