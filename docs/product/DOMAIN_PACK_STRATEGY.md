# Domain Pack Strategy

## Strategy

Use domain packs to prove Martenweave in real scenarios without narrowing the core product.

## Pack Contents

A domain pack may include:

- starter model templates;
- example repositories;
- optional validation rules;
- domain-specific metadata conventions;
- demo scripts and documentation.

## Core Boundary

Domain packs must not:

- define core object semantics;
- make domain-specific fields mandatory in generic validation;
- add platform dependencies;
- introduce direct writes to business systems.

## Starter Packs

Current starter directions:

- generic product model for domain-neutral simple table mode;
- SAP Business Partner / Customer model for enterprise domain-pack validation;
- future analytics, finance, procurement, HR, or custom enterprise models.
