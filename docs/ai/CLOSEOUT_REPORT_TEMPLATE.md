# Closeout Report Template

Use this format after completing an issue or task.

```markdown
## Result
Completed / Partially completed / Blocked

## Summary
- What changed.
- Why it satisfies the goal.

## Files Changed
- `path`: reason.

## Validation
- `command`: pass/fail and short result.

## Generated Artifacts
- None committed / intentionally changed with reason.

## Core vs Domain-Pack Impact
- Core impact:
- Domain-pack impact:

## Safety Notes
- Secrets/data/generated-file status.
- Human approval required, if any.

## Follow-Up
- Issues created or recommended.
```
